🖥️ A. Redes IT Tradicionales Information Technology
Definición ▶
Infraestructura de comunicación para procesar y transmitir datos entre computadoras y servidores.
Propósito principalConectar computadoras, servidores y dispositivos de usuario para compartir recursos e información.
Entorno típicoOficinas, centros de datos, universidades, empresas.
Componentes principales ▶
Equipos físicos y lógicos que forman la infraestructura.
RoutersEnrutan paquetes entre diferentes redes, conectan LAN con WAN/Internet.
SwitchesConmutan tramas en la capa 2, interconectan dispositivos en la misma LAN.
FirewallsControlan el tráfico y protegen perímetros de red.
ServidoresDNS, DHCP, web, correo, bases de datos.
Cableado / Wi-FiEthernet Cat5e/6/7, fibra óptica, 802.11 Wi-Fi.
Protocolos comunes ▶
Estándares de comunicación utilizados.
TCP/IPProtocolo base de Internet y redes empresariales.
HTTP / HTTPSTransferencia de páginas web y APIs.
FTP / SFTPTransferencia de archivos.
SMTP / IMAP / POP3Correo electrónico.
DNS / DHCPResolución de nombres e IPs dinámicas.
BGP / OSPFProtocolos de enrutamiento dinámico.
Características ▶
Atributos principales de las redes IT.
Alta velocidadDesde 100 Mbps hasta 100 Gbps o más en backbone.
Baja latenciaDiseñadas para respuesta en tiempo real para usuarios.
Seguridad robustaVPNs, cifrado TLS, autenticación 802.1X, segmentación VLAN.
Administración centralizadaSNMP, Syslog, SDN, herramientas de monitoreo.
Escalabilidad planificadaArquitecturas jerárquicas: Core, Distribución, Acceso.
Topologías DISEÑO ▶
Configuraciones físicas y lógicas de la red.
EstrellaTodos los nodos conectados a un switch/hub central.
Árbol jerárquicoCapas Core → Distribución → Acceso.
MallaAlta redundancia, usada en WAN y centros de datos.
AnilloToken Ring, SONET/SDH en fibra óptica.
📡 B. Redes IoT Internet of Things
Definición ▶
Red de objetos físicos con sensores y conectividad para intercambiar datos con sistemas y otros dispositivos.
Propósito principalConectar "cosas" del mundo físico: sensores, actuadores, electrodomésticos, vehículos, maquinaria industrial.
Entorno típicoHogares inteligentes, fábricas, ciudades inteligentes, salud, agricultura.
Dispositivos ▶
Tipos de endpoints conectados en IoT.
SensoresTemperatura, humedad, presión, luz, movimiento, GPS.
ActuadoresMotores, válvulas, relés controlados remotamente.
Gateways IoTConcentran datos de múltiples dispositivos y los envían a la nube.
Edge devicesProcesan datos localmente para reducir latencia.
WearablesSmartwatches, monitores de salud, rastreadores fitness.
Protocolos IoT ▶
Estándares especializados para dispositivos con recursos limitados.
MQTTMessage Queuing Telemetry Transport — ligero, pub/sub, ideal para sensores.
CoAPConstrained Application Protocol — REST sobre UDP para redes con pérdidas.
Zigbee / Z-WaveRedes de malla de baja potencia para domótica.
LoRaWANLargo alcance, ultra baja potencia para ciudades inteligentes y agricultura.
BLEBluetooth Low Energy — wearables y beacons.
NB-IoT / LTE-MIoT sobre redes celulares, cobertura amplia.
AMQPProtocolo de mensajería para IoT empresarial.
Arquitectura IoT ▶
Capas de la arquitectura típica IoT.
Capa de percepciónSensores y actuadores que interactúan con el mundo físico.
Capa de red / transporteTransmisión de datos vía Wi-Fi, celular, LoRa, Zigbee.
Capa de procesamientoEdge computing, fog computing, gateways locales.
Capa de aplicaciónPlataformas cloud (AWS IoT, Azure IoT Hub, Google IoT Core), dashboards, analytics.
Desafíos IoT RETOS ▶
Problemáticas específicas del ecosistema IoT.
SeguridadSuperficie de ataque masiva, dispositivos con recursos limitados para cifrado.
InteroperabilidadMúltiples estándares y fabricantes incompatibles.
EscalabilidadMillones o billones de dispositivos conectados simultáneamente.
Gestión del ciclo de vidaActualizaciones OTA (Over The Air) en dispositivos distribuidos.
PrivacidadRecolección masiva de datos personales y de comportamiento.
⚡ C. Diferencias IT vs IoT Comparativa
CRITERIO 🖥️ REDES IT 📡 REDES IoT
Dispositivos
Computadoras, servidores, laptops, smartphones
Sensores, actuadores, microcontroladores, dispositivos embebidos
Recursos hw
Alta capacidad: procesadores potentes, mucha RAM y almacenamiento
Muy limitados: CPU de MHz, KB de RAM, almacenamiento mínimo
Consumo energía
Alto, conectados a corriente eléctrica permanente
Ultra bajo, frecuentemente alimentados por baterías (años)
Protocolos
TCP/IP, HTTP, FTP, SMTP, DNS — protocolos estándar de Internet
MQTT, CoAP, Zigbee, LoRa — ligeros y eficientes para restricciones
Conectividad
Ethernet Gbps, Wi-Fi 802.11ac/ax, fibra óptica
LoRaWAN, NB-IoT, BLE, Zigbee — alcance y consumo optimizados
Escala
Miles de dispositivos por empresa (manejable)
Millones o billones de dispositivos globales
Seguridad
Firewalls, VPN, TLS, autenticación multifactor, IDS/IPS
Limitada por hw, firmas de código, cifrado ligero, mayor vulnerabilidad
Ancho de banda
Alto: transmisión de video, archivos grandes, aplicaciones cloud
Bajo: pequeños paquetes de telemetría periódicos
Latencia
Baja y predecible, crítica para apps interactivas
Variable, tolerable en muchos casos (excepto IoT crítico)
Administración
Centralizada, con administradores de red dedicados
Automatizada, OTA updates, plataformas de gestión masiva
Aplicación
ERP, correo, web, videoconferencia, bases de datos
Smart home, industria 4.0, salud conectada, smart cities
🔗 D. Capas del Modelo TCP/IP y Funciones Principales 4 Capas
CAPA 1 — INFERIOR
Acceso a Red
(Network Access / Link)
Ethernet Wi-Fi 802.11 ARP PPP HDLC Frame Relay
Funciones: • Transmisión física de bits en el medio (cable, fibra, radio)
• Direccionamiento físico mediante MAC
• Detección y corrección de errores de enlace (CRC)
• Control de acceso al medio (CSMA/CD, CSMA/CA)
• Encapsulación en tramas
CAPA 2
Internet
(Network Layer)
IPv4 IPv6 ICMP OSPF BGP RIP
Funciones: • Direccionamiento lógico mediante IP (IPv4/IPv6)
• Enrutamiento: selección de la mejor ruta entre redes
• Fragmentación y reensamblado de paquetes
• TTL (Time to Live) para evitar bucles infinitos
• ICMP: diagnóstico (ping, traceroute)
CAPA 3
Transporte
(Transport Layer)
TCP UDP SCTP Puertos
Funciones: • TCP: conexión fiable, control de flujo, retransmisión, 3-way handshake
• UDP: sin conexión, rápido, para streaming/VoIP/juegos
• Multiplexación mediante puertos (0–65535)
• Segmentación y reensamblado de datos
• Control de congestión de red
CAPA 4 — SUPERIOR
Aplicación
(Application Layer)
HTTP/S DNS SMTP FTP SSH DHCP SNMP MQTT
Funciones: • Interfaz directa entre red y aplicaciones de usuario
• DNS: resolución de nombres de dominio a IPs
• DHCP: asignación dinámica de IPs
• Transferencia web (HTTP), correo (SMTP), archivos (FTP)
• Gestión remota (SSH, SNMP, Telnet)
